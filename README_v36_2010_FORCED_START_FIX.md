# Magic Split v36 - 2010 장기검증 시작일 강제 고정 패치

## 문제
v35에서는 최대 검증 거래일 입력 상한을 5000으로 올렸지만, Streamlit 세션에 기존 `bt_start_date` 값이 남아 있으면 `2010부터 장기검증`을 선택해도 실제 시작일이 2014년 부근으로 유지될 수 있었습니다.

즉 숫자 입력을 5000으로 바꿔도, 시작일 입력값이 예전 최근 3000거래일 기준 날짜를 물고 있어서 2010 백테스트가 되지 않는 문제가 있었습니다.

## 수정
v36에서는 장기검증 프리셋을 선택하면 시작일을 프리셋 날짜로 강제 고정합니다.

- `2010부터 장기검증`: 2010-01-04 고정
- `2020부터 전체검증`: 2020-01-02 고정
- `2021부터 전체검증`: 2021-01-04 고정

또한 Streamlit 세션 충돌을 피하기 위해 key를 프리셋별로 분리했습니다.

- `bt_start_date_v36_<프리셋>`
- `bt_end_date_v36_<프리셋>`
- `bt_max_days_v36_5000_<프리셋>`

## 확인 방법
앱 실행 후:

1. `6. 섹터전략 백테스트`
2. `백테스트 기간`에서 `2010부터 장기검증` 선택
3. 시작일 칸이 입력창이 아니라 `2010-01-04` metric으로 표시되는지 확인
4. 최대 검증 거래일은 기본 4300, 최대 5000까지 입력 가능
5. `T10/T100 Turbo 공격ETF 100% NO CTA` 선택 후 실행

## 주의
2010 장기검증은 ETF 상장일 제약 때문에 완전 동일 실전형이 아니라 데이터가 있는 자산부터 자동 편입되는 장기 프록시 검증입니다.
