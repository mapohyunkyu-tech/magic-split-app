# Magic Split v39 - T100 2010 프록시 확장 백테스트

## 목적
T10/T100 Turbo NO CTA 2010 백테스트에서 2010~2014 상반기가 CASH 100%로 남는 원인을 확인하고, ETF 실제 조회 데이터 기준과 지수/환율 프록시 확장 기준을 분리합니다.

## 확인된 원인
업로드된 실제 ETF 기준 CSV에서는 앱이 확보한 KODEX200/NASDAQ100/BOND 데이터가 2014-04-08부터 시작했습니다. 모멘텀 최소 계산 구간 때문에 첫 비현금 진입은 2014-07-01이었습니다.

## v39 추가 기능
- `T10/T100 Turbo 공격ETF 100% NO CTA - 지수/환율 프록시` 옵션 추가
- 프록시 로더 추가
  - KODEX200 → KOSPI200 지수
  - KOSDAQ150 → KOSDAQ150 또는 KOSDAQ 지수
  - NASDAQ100 → NASDAQ100/QQQ + USDKRW
  - GOLD → 금 선물/금 가격 + USDKRW
  - DOLLAR → USDKRW
- summary에 `자산별데이터기간`, `자산별사용가능일수`, `백테스트데이터방식` 추가

## 사용법
1. `app.py` 교체
2. Streamlit 재시작
3. `6. 섹터전략 백테스트`
4. 기간: `2010부터 장기검증`
5. 방공호 방식: `T10/T100 Turbo 공격ETF 100% NO CTA - 지수/환율 프록시`
6. `방공호/Turbo만 빠른 실행` 클릭

## 주의
이 프록시 결과는 ETF 실제 상장일 기준 백테스트가 아니라 장기 지수/환율 확장 검증입니다. 실제 매수 가능성·추적오차·보수·세금·환헤지 차이는 별도 고려해야 합니다.
