# v59 T100 HYBRID 1↔3 LIVE RESET / EXPERIMENT FIX

## 수정 내용

운용모드 입력값이 Streamlit 세션에 저장되어 `현재 총자산 1,600만원 / CASH 600만원`이 계속 살아나고, 그 결과 6310 목표/판정이 올라가는 문제를 수정했습니다.

### 핵심 변경

- 기존 v54/v55 입력 키와 완전히 분리해 v59 새 키 사용
- 기본값을 천만원 실험 기준으로 고정
  - 현재 총자산: 10,000,000
  - 현재 T100 평가금액: 10,000,000
  - 현재 CASH/예수금: 0
  - 직전 잠금 기준금액: 10,000,000
  - 현재 운용모드: 1순위 공격모드
- `천만원 실험값으로 초기화` 버튼 추가
- `T100 + CASH`가 현재 총자산과 다르면 경고 표시
- Streamlit API 도움말표 출력 방지 설정 유지

## 천만원 실험 입력법

천만원만 실험할 경우 아래처럼 입력합니다.

```text
현재 총자산: 10,000,000
현재 T100 평가금액: 10,000,000
현재 CASH/예수금: 0
직전 잠금 기준금액: 10,000,000
현재 운용모드: 1순위 공격모드
이미 6310 잠금에 진입한 계좌: 체크 안 함
```

이때 6310 전환 목표는 15,000,000원이며, 현재는 6310 불가로 나와야 정상입니다.

## 600만원이 추가입금인 경우

600만원을 같은 계좌에 추가로 넣은 것이라면 앱이 수익으로 오해하지 않도록 잠금 기준도 같이 올려야 합니다.

```text
현재 총자산: 16,000,000
현재 T100 평가금액: 10,000,000
현재 CASH/예수금: 6,000,000
직전 잠금 기준금액: 16,000,000
```

## 실행

```bash
streamlit run app.py
```

권장 실행 방식:

1. ZIP 압축을 풉니다.
2. 압축 푼 폴더 안에서 실행합니다.
3. 기존 브라우저 창을 닫고 새로 엽니다.
4. 입력값이 이상하면 `천만원 실험값으로 초기화` 버튼을 누릅니다.
