# v60 T100 HYBRID 1↔3 운용모드 - 예수금 메모 분리 패치

## 수정 이유
사용자가 입력한 의미는 다음과 같음.

- 운용대상 투자금액: 10,000,000원
- 현재 T100 평가금액: 10,000,000원
- 계좌에 보이는 예수금: 6,000,000원

기존 앱은 예수금 600만원을 전략 CASH로 보고 T100 실험자금에 포함하여 목표금액/6310 조건/리밸런싱 금액이 꼬일 수 있었다.

## v60 수정 내용

운용모드 입력값을 분리함.

1. 운용대상 투자금액
   - T100 실험에 실제로 쓰는 금액만 입력
   - 천만원 실험이면 10,000,000

2. 현재 T100 평가금액
   - 현재 T100 ETF 평가금액

3. 전략 CASH/방어현금(계산 포함)
   - CAP5 방어, 6310 잠금으로 만든 현금만 입력
   - 목표비중과 리밸런싱 계산에 포함

4. 계좌 예수금 메모(계산 제외)
   - 계좌에 보이는 예수금이지만 T100 실험 자금이 아니면 여기에 입력
   - 목표금액, 6310 조건, 리밸런싱 계산에서 제외

## 천만원 실험 입력 예시

- 운용대상 투자금액: 10,000,000
- 현재 T100 평가금액: 10,000,000
- 전략 CASH/방어현금: 0
- 계좌 예수금 메모: 6,000,000
- 직전 잠금 기준금액: 10,000,000
- 현재 운용모드: 1순위 공격모드

이 경우 앱은 예수금 600만원을 목표매수에 사용하라고 하지 않는다.
