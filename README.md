# MAGIC V27 SECTOR AUTO CONFIDENCE UPGRADE

v27 기반 섹터신뢰 자동 보강판입니다.

## 핵심 변경

- 사용자가 매번 직접 섹터를 고치지 않아도 되도록 `KIND/FDR 업종·주요제품 + 내장 키워드` 자동 보정 로직을 추가했습니다.
- Google Sheets `섹터매핑` 탭은 선택 보정값으로 유지합니다. 값이 있으면 최우선, 없어도 자동 분류가 동작합니다.
- TOP50과 보유차수 판단기가 동일한 `infer_sector_profile()` 섹터 엔진을 사용합니다.
- 신규 TOP50은 `섹터신뢰 낮음`이면 점수상한 74.9 및 최종판정 최대 `대기`가 적용됩니다.
- 보유차수 판단기는 `섹터신뢰 낮음`만으로 요양원 처리하지 않고, 손익률/차수/거래대금/점수 기준으로 판단합니다.
- 화면 메뉴 `5. 섹터진단`에서 자동 원천 수집 개수, KIND/FDR 원천 개수, 테스트 종목 섹터판정을 확인할 수 있습니다.

## 자동 섹터 우선순위

1. Google Sheets `섹터매핑` 값이 있으면 최우선
2. 내장 코드확정맵
3. KIND 상장법인목록 업종/주요제품
4. FinanceDataReader `KRX-DESC`/`KRX` 업종 정보
5. 종목명/사업 키워드 보정
6. 그래도 실패하면 `기타 / 낮음 / 수동확인필요`

## 배포 방법

GitHub 루트에 아래 파일을 올리세요.

- `app.py`
- `requirements.txt`
- `runtime.txt`
- `README.md`

## 확인 방법

1. Streamlit 실행 후 좌측 메뉴에서 `5. 섹터진단`으로 이동
2. `자동 업종 원천 종목` 수가 0보다 큰지 확인
3. 테스트 입력에 `기가비스`, `엘티씨`, `삼성E&A`, `HMM`, `S-Oil` 등을 넣어 확인
4. TOP50 생성 후 `섹터분류`, `섹터신뢰`, `섹터판정` 컬럼 확인

## 주의

- Streamlit 서버에서 KIND/FDR 원천 호출이 일시 실패하면 자동 원천 개수가 0일 수 있습니다. 이 경우 내장 코드맵과 키워드 보정만으로 동작합니다.
- `섹터신뢰 보정`은 공식 업종/주요제품 또는 키워드 기반 자동 추론입니다. 완전 확정은 내장 코드확정맵 또는 `섹터매핑` 값일 때만 부여합니다.
- KRX/KIND/FDR 원천은 네트워크 상태나 원천 구조 변경에 따라 실패할 수 있도록 안전하게 빈 데이터로 폴백됩니다.
