# v54 T100 HYBRID 1↔3 LOCK 운용모드

## 목적

v53은 `T100 HYBRID 1↔3 LOCK`을 백테스트 메뉴에 붙인 버전이었습니다.
이번 v54는 백테스트가 아니라 실제 운용판에서 오늘 기준으로 다음을 판단하도록 추가한 버전입니다.

- 현재 1순위 공격모드인지
- 1순위 방어모드로 낮춰야 하는지
- 3순위 6310 잠금모드로 전환해야 하는지
- 현재 T100/CASH 금액 대비 얼마를 늘리거나 줄여야 하는지
- 오늘 기준 T100 내부 선택자산이 무엇인지

## 추가 메뉴

```text
7-1. T100 하이브리드 운용모드
```

## 운용 규칙

### 1순위

```text
T100 CAP5 + 5일 누적 -6% 방어
평상시: T100 100%
위험 신호: T100 70% + CASH 30%
```

위험 신호:

```text
하루 수익률 <= -5%
또는
최근 5거래일 누적 수익률 <= -6%
```

### 3순위

```text
T100 SPLIT 60/30/10 + C안
운용판 계산: T100 60% + CASH 40%
```

CASH 40%는 아래를 합산한 것으로 봅니다.

```text
대기현금 30% + 생활비/잠금현금 10%
```

### 1↔3 전환

```text
처음: 1순위 공격모드
직전 잠금 기준 대비 +50% 달성: 3순위 6310 잠금모드 진입
6310 진입 후 최소 60거래일 유지
T100이 20일선 위 + 고점 대비 -5% 이내 회복: 1순위 복귀
```

## 리밸런싱 기준

```text
T100 내부 자산선택: 월 1회
CAP5 / 5일 누적 위험 신호: 매일 체크
1↔3 전환: +50% 수익잠금 이벤트 때만 체크
```

## 입력값

- 현재 총자산
- 현재 T100 평가금액
- 현재 CASH/예수금
- 직전 잠금 기준금액
- 현재 운용모드
- 6310 진입일
- CAP5 기준
- 5일 누적손실 기준
- 1순위 방어 시 T100 노출
- 3순위 6310 T100 노출
- 6310 최소 유지일

## 출력값

- 오늘판정
- 목표모드
- 목표 T100 노출
- 목표 CASH 비중
- 오늘 목표 자산배분
- 현재금액 대비 리밸런싱 금액
- T100 월간 선택자산
- 최근 30거래일 T100 신호
- 최근 12개월 월간 선택자산
- 데이터 상태

## 사용 예시

천만원 실험 기준:

```text
현재 총자산: 10,000,000
현재 T100 평가금액: 10,000,000
현재 CASH/예수금: 0
직전 잠금 기준금액: 10,000,000
현재 운용모드: 1순위 공격모드
```

계좌가 1,500만원 이상으로 커지면:

```text
직전 잠금 기준금액 10,000,000 대비 +50% 조건 충족
→ 3순위 6310 잠금모드 전환
→ 목표 T100 60%, CASH 40%
```
