# Magic Split v27 - BUNKER 70/30 DUAL FIX (2026-06-30)

## 핵심 수정

기존 70/30 방공호 백테스트에서 `월간 듀얼모멘텀 상위2`를 선택해도 결과가 전 기간 `CASH:100%`로만 찍히는 문제가 있었습니다.

이번 수정판은 다음을 보정했습니다.

- 방공호 ETF 가격 로딩을 FDR 원본 `Close` 컬럼 우선으로 변경
- ETF 데이터 로딩 실패 시 조용히 현금으로 대체하지 않고 요약에 진단값 표시
- 듀얼모멘텀 초기 구간에서 200일 데이터 부족 시 MA120/MA60/MA45 순서로 완화
- 방공호 일별 CSV에 `방공호데이터자산수`, `방공호데이터자산`, `방공호모드` 추가
- 방공호 요약 CSV에 `방공호가격데이터자산수`, `방공호가격데이터자산`, `CASH100일수`, `비현금방공호일수`, `마지막방공호보유`, `방공호데이터상태` 추가
- 듀얼 선택 후 전 기간 CASH:100%이면 화면에 오류 표시

## 정상 확인 기준

`magic_split_bunker_7030_daily_*.csv`의 `방공호보유`가 아래처럼 날짜에 따라 바뀌어야 정상입니다.

```text
NASDAQ100:50%,GOLD:50%
GOLD:50%,DOLLAR:50%
KODEX200:50%,NASDAQ100:50%
CASH:100%
```

전 기간 `CASH:100%`이면 듀얼이 적용된 것이 아니라 데이터 로딩 실패 또는 모멘텀 조건 전부 미충족입니다.
