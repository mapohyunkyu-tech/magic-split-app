# v75 T100 추가입금 보정 방어판정

## 핵심 수정

`7-1. T100 하이브리드 운용모드`에서 방어판정 표기를 다음처럼 분리했습니다.

- T100 누적 투입원금: 실제 전략에 넣은 돈의 누적. 추가입금/인출할 때만 바뀜.
- 어제 평가금액: 전일 장마감 T100 평가금액.
- 오늘 장마감 T100 평가금액: 오늘 실제 평가금액.
- 오늘 전략 추가입금/인출: 투입원금 증감용.
- 오늘 T100 실제 추가매수액: 오늘 ETF를 실제로 더 산 금액.
- 오늘 T100 현금화매도액: 방어/6310 전환 등으로 T100에서 현금화한 금액.

## 방어판정 계산

1일 -5% 방어판정은 투입원금 기준이 아니라 아래 기준입니다.

```text
오늘 판정기준금액 = 어제 T100 평가금액 + 오늘 T100 실제 추가매수액 - 오늘 T100 현금화매도액
1일 변동률 = 오늘 장마감 T100 평가금액 / 오늘 판정기준금액 - 1
```

예시:

```text
어제 T100 평가금액 1,000만
오늘 T100 실제 추가매수액 500만
오늘 판정기준금액 1,500만
```

장마감 평가금액이:

```text
1,560만 = +4% → 방어 OFF
1,440만 = -4% → 방어 OFF
1,380만 = -8% → 방어 ON
```

## 방어날 추가입금

방어모드에서 900만을 전략에 넣더라도 전액을 사지 않고 70/30을 맞춰 일부만 T100에 매수할 수 있습니다.
이때는 다음처럼 입력합니다.

- 오늘 전략 추가입금: 900만
- 오늘 T100 실제 추가매수액: 실제 ETF 산 금액만 입력
- 나머지 방어현금은 계좌 예수금 참고/전략 현금으로 관리

그러면 새 돈 때문에 손실률이 희석되어 방어판정이 꺼지는 문제를 막을 수 있습니다.

## 5일 누적 -6%

v75부터 5일 누적은 최근 저장된 `1일변동률`을 누적해서 계산합니다.
추가입금이 있어도 `T100 실제 추가매수액`으로 보정하기 때문에 추가입금이 수익으로 잡히지 않습니다.

## 유지된 기능

- v68의 2010 장기검증 / 방공호·Turbo 빠른 실행 유지
- v74의 홈 폴더 고정 저장 유지
- 기존 T100 운용기록 CSV 자동 보정
- 기존 10번 A분할 백테스트 메뉴는 남겨두었지만, 실전 메인은 7-1 T100 단순 운용모드입니다.
