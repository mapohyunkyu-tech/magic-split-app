# MAGIC v27 MANUAL LOT PARSER FIX

## 수정명
v27_MANUAL_LOT_PARSER_FIX_20260617

## 수정 이유
보유차수 판단기 > 수동 추가에서 아래 형식 입력 시 종목 헤더가 정상 인식되지 않으면,
차수 줄의 첫 값 `1`, `2`를 종목명처럼 검색하여 `못 찾은 종목: 1, 2`가 뜨는 문제가 있었습니다.

예시 입력:

```text
DB하이텍
1, 193300, 1
2, 174200, 1
```

기존 오류:

```text
추가된 차수 0개
못 찾은 종목: 1, 2
```

## 수정 내용
1. 종목명 한 줄 + 아래 차수 입력 방식 안정화
2. `[종목명]` 방식 유지
3. 기존 `종목명,차수,진입단가,수량` 한 줄 방식 유지
4. 종목 헤더를 못 찾았을 때 차수 숫자 `1`, `2`를 종목명으로 검색하지 않도록 수정
5. 못 찾은 종목 표시가 차수 숫자가 아니라 실제 헤더 종목명으로 나오도록 수정
6. DB하이텍 수동 코드맵 추가
   - DB하이텍 / 디비하이텍 / DB하이텍 000990 → 000990 DB하이텍

## 정상 입력 예시

```text
DB하이텍
1, 193300, 1
2, 174200, 1
```

또는

```text
[DB하이텍]
1,193300,1
2,174200,1
```

또는

```text
DB하이텍,1,193300,1
DB하이텍,2,174200,1
```
