# Magic Split v34 - 2010 장기검증 프록시 모드

## 핵심 변경

`6. 섹터전략 백테스트` 메뉴의 기간 프리셋에 `2010부터 장기검증`을 추가했습니다.

- 시작일 기본값: 2010-01-04
- 최대 검증 거래일 상한: 3000 → 5000
- 기본 검증 거래일: 4300
- T10/T100 Turbo NO CTA 장기검증 우선 사용

## 해석 주의

2010부터는 모든 ETF가 같은 시점에 존재하지 않습니다.
따라서 v34의 2010 검증은 완전 동일 실전 ETF 백테스트가 아니라,
데이터가 있는 자산부터 자동 편입되는 장기 프록시 검증입니다.

- KODEX200은 2010 전체 구간 사용 가능
- KOSDAQ150 ETF는 상장 이후부터 사용 가능
- 국내 NASDAQ100 ETF는 상장 이후부터 사용 가능
- CTA 포함형은 DBMF/KMLM/CTA 상장 이후부터 CTA가 반영됨
- T100/T10 NO CTA가 2010 장기검증 1순위

## 사용법

1. 메뉴: `6. 섹터전략 백테스트`
2. 백테스트 기간: `2010부터 장기검증`
3. 방공호 방식: `T10/T100 Turbo 공격ETF 100% NO CTA`
4. 최대 검증 거래일: 기본 4300 또는 5000까지 가능
5. 실행 후 summary CSV에서 `2010장기검증주의` 확인
